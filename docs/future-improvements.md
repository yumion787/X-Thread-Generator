# 今後のアップデート候補

機能拡張・UI改善・品質向上のアイデアをカテゴリ別に整理。

---

## 機能拡張

### 生成機能
- [ ] **プロンプトテンプレート選択**: ターゲット層（ビジネス / テック / ライフスタイル等）を選択できるようにする
- [ ] **投稿数の指定**: 5〜10ポストの中からユーザーが件数を指定できるようにする
- [ ] **再生成ボタン**: 生成結果が気に入らない場合、同じURLで再度生成できるボタンを追加
- [ ] **Xへの直接投稿**: X（Twitter）API連携で生成したスレッドをそのまま投稿

### ユーザー管理
- [ ] **サブスクリプションプラン**: 月額制プランの追加（使い放題など）
- [ ] **クレジット残高通知**: 残数が少なくなったときのメール通知

### 履歴機能
- [ ] **履歴の検索・フィルター**: URL・日付・キーワードで絞り込み
- [ ] **お気に入り登録**: よく使うスレッドをピン留め
- [ ] **50件以上の表示**: ページネーション or 無限スクロール対応

---

## UI / UX 改善

### 生成画面
- [ ] **ストリーミング表示**: 生成結果をリアルタイムで逐次表示（現在は完了後に一括表示）
- [ ] **生成プレビュー**: Xのスレッド表示に近いプレビューモード

### オンボーディング
- [ ] **初回ガイド**: 初回ログイン時にサービスの使い方を案内するツールチップ or ガイド

### モバイル
- [ ] **PWA対応**: ホーム画面に追加できるようにする

## 品質・保守

### エラー・モニタリング
- [ ] **Sentry導入**: フロントエンド / サーバーサイドのエラー監視
- [ ] **ログ整理**: Webhook等の console.log を本番では出力しない

### パフォーマンス
- [ ] **APIレート制限**: ユーザー / IP単位でのスパム防止

### セキュリティ
- [ ] **Content Security Policy**: CSPヘッダーの設定
- [ ] **入力値サニタイズ**: URLの追加バリデーション強化
