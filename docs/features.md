# 機能一覧

## 1. スレッド自動生成

**場所**: トップページ（ログイン済み時）
**コンポーネント**: `components/Generator.tsx`
**API**: `POST /api/generate-thread`

### フロー

```
URL入力 → 記事スクレイピング（Cheerio）→ クレジット消費 → Claude API → 結果表示
```

### 処理詳細

1. URLバリデーション（クライアント + サーバー両方）
2. クレジット残高チェック（消費前に確認）
3. 記事取得: `<article>` → `<main>` → `<body>` の優先順でテキスト抽出
   - `script` / `style` / `nav` / `header` / `footer` / `aside` / `.sidebar` / `.ads` を除去
   - 空白を圧縮し最大8,000文字に制限
4. 記事の最低文字数チェック（200文字未満はエラー）
5. **クレジットを消費**（`consume_credit` RPC）
6. Claude API で5〜10ポストを生成（1ポスト=140文字以内）
7. 生成失敗時はクレジットを返還
8. 結果を `threads` テーブルに保存
9. 生成結果と残クレジットを返却

### エラーハンドリング

| 状況 | メッセージ | クレジット消費 |
|------|-----------|--------------|
| 認証なし | ログインが必要です | なし |
| URL不正 | 無効なURLです | なし |
| ページ404 | ページが見つかりませんでした（404） | なし |
| ページ403 | アクセスが拒否されました（403） | なし |
| 接続失敗 | URLへの接続に失敗しました | なし |
| 文字数不足 | 記事の内容が不足しているため〜 | なし |
| クレジット不足 | クレジットが不足しています | なし |
| AI失敗 | AI生成中にエラーが発生しました | **返還** |
| JSON解析失敗 | AIの応答を解析できませんでした | **返還** |
| AI不適切判定 | ブログ記事として認識できる内容が〜 | **返還** |

### 生成プロンプト

- **ターゲット**: 20〜30代・キャリアやお金に不安を感じている社会人
- **フック**: 逆説や具体的な数字で始める
- **構成**: フック → 本文（1ポスト1トピック）→ まとめ → URL誘導 + ハッシュタグ
- **形式**: JSON `{"posts": [{"index": N, "content": "..."}]}`

## 2. スレッドの編集・コピー

**コンポーネント**: `components/EditablePost.tsx`

- 生成された各ポストを **インラインで編集可能**（textarea）
- 140文字カウンター（超過時は赤色警告）
- ポスト単体のコピーボタン
- 全ポスト一括コピーボタン（`\n\n` 区切り）

---

## 3. 認証（Google OAuth）

**コンポーネント**: `components/LoginModal.tsx` / `contexts/AuthContext.tsx`
**API**: `GET /auth/callback`

- Googleアカウントでのワンクリックログイン
- 未ログイン状態でURL入力 → ログイン → URL自動復元（`localStorage` 経由）
- ログアウト時は確認ダイアログを表示し `/` にリダイレクト

### URL保持フロー

```
未ログインでURL入力 → localStorage に保存 → LoginModal 表示
→ Google認証 → /auth/callback → / にリダイレクト
→ SearchParamsHandler が localStorage を検出 → URL自動セット
```

---

## 4. クレジットシステム

**コンポーネント**: `components/CreditBadge.tsx`
**DB関数**: `consume_credit` / `add_credits` RPC

- ヘッダーに残クレジット数を常時表示（md以上の画面）
- 残数≤1で赤色バッジ + 購入ボタン表示
- 新規登録時に **3クレジット自動付与**（Supabase トリガー）
- 生成成功後にクレジット表示を即時更新（`refreshCredits()`）
- クレジット消費はRPCで **アトミック処理**（競合防止）

---

## 5. Stripe決済

**API**: `POST /api/stripe/checkout` / `POST /api/stripe/webhook`

- 10クレジット ¥300（税込）の一回払い
- Stripeホスト決済ページへリダイレクト
- 決済完了後、Webhookでクレジットを即時付与
- 決済成功 / キャンセル後はトーストで通知

---

## 6. 生成履歴

**ページ**: `/history`
**コンポーネント**: `app/history/HistoryClient.tsx`

- 直近50件の生成履歴を一覧表示
- 各履歴カードの情報: ホスト名・URL・生成日時・ポスト数
- **アコーディオン展開**でポスト一覧を表示（インライン編集可能）
- **「このURLで再生成」ボタン**: アウトラインボタンで当該URLを再生成
- **削除機能**: 確認ダイアログ → Supabaseから削除 → 楽観的UI更新
- 未ログイン時は `/` にリダイレクト（サーバーコンポーネントで処理）

## 7. ランディングページ（未ログイン時）

**コンポーネント**: `components/LandingPage.tsx`

## 8. ヘッダー

**コンポーネント**: `components/Header.tsx`
